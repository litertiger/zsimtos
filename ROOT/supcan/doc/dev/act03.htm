3.集成部署

　　所谓“集成部署”是指将硕正套件集成到您开发的 Web 应用中、连同您的应用一起发布。
　　解开 zip 包，会发现里面除了大量演示用的 Html 页面和静态的 xml 数据，真正的核心都在 binary 目录中，这些文件用途说明如下：
文件名用途文件尺寸
核心包supcan2.cab用于浏览器的加载项120k
supcan.crx用于浏览器的Plug-in插件120k
supcan.xpi用于浏览器的Plug-in插件110k
BCV1.bin硕正树列表、自由表头等基础功能组件900k
LuxForm.bin硕正报表组件560k
License.bin授权证书包1k
可选包BCV4.bin硕正Freeform设计器100k
gzip.bin让硕正套件能读取后端gzip压缩后的数据
注：首次使用时能自动下载		16k
libxl.bin硕正已购买的第三方程序包, 用于导出成 xls 格式、从 xls 文件导入(硕正企业版)
注：首次使用时随 msoffice.bin 一起下载		1.2M
msoffice.bin用于读写 Excel 2007 的 xlsx 格式文件
注：首次使用时能自动下载		60k
LuxFormC.bin用于Report的上报汇总模式
注：首次使用时能自动下载		25k
WebFile.bin用于文件的上传下载
注：首次使用时能自动下载		30k
barcode.cab用于显示条形码
注1.条形码功能不支持64位浏览器;
注2.该文件不在zip包中, 需要您单独到 www.supcan.cn 下载区的 "其它第三方功能组件" 中去下载, 再将其部署到binary目录;
注3.首次使用时能自动下载，但很难完成自动安装，原因不明，如果安装失败，解开原始zip包，执行 exe 文件安装；
1.1M
64位IE包supcan2.x64.cab用于 64 位 IE 浏览器的加载项140k
同名的 *.x64.bin用于 64 位 IE 浏览器约4M


　　部署工作很简单：只要将 binary 子目录复制到您的 Web 虚拟目录即可。

　　此外，如果您在页面中需要使用硕正报表，那么还进一步需要将 doc 子目录也复制过去，doc 子目录主要是一些在线帮助的内容，最终用户在使用硕正套件过程中可能需要浏览。

doc 须整个子目录拷贝走，相对目录须 bin 文件所在的子目录并列.


部署到其它服务器时，别忘了配置 MIME 类型.
